# Discord Bot コマンド仕様書

## 共通仕様

- コマンドはすべて **スラッシュコマンド** (`/`) として実装する
- Bot の応答はすべて **Ephemeral（本人のみ表示）** を基本とする  
  ただし `/post-game` など明示的に全員へ投稿するものは除く
- 管理者コマンドは Discord の **ロール** で制御する（サーバー設定で管理者ロールを指定）
- Webサーバー API を通じてデータを取得・更新する

---

## コマンド一覧

| コマンド | 説明 | 権限 |
|---------|------|------|
| `/bet` | ゲームに賭けを行う | 全員 |
| `/mybet` | 指定ゲームの自分の賭け状況を確認する | 全員 |
| `/mybets` | 開催中イベント内で自分が賭けた全ゲームの状況を確認する | 全員 |
| `/admin-link` | 管理者用 WebアプリのURLを表示する | 管理者のみ |
| `/post-game` | ゲーム情報をチャンネルに投稿する | 管理者のみ |

## `/bet` - 賭けを行う

### 説明
現在開催中のゲームに対してポイントを賭ける。

### オプション

| オプション名 | 型 | 必須 | 説明 |
|------------|-----|------|------|
| `game` | Integer | ○ | ゲームID（ゲーム番号） |
| `option` | String | ○ | 賭ける記号の文字列。ゲームの賭け方式に応じた入力が必要（後述） |
| `amount` | Integer | ○ | 賭けるポイント数（1以上の整数） |
| `borrow` | Boolean | - | 借金として賭けるか（デフォルト: `false`） |

#### `option` の入力形式（賭け方式別）

| 賭け方式 | 入力例 | 説明 |
|----------|--------|------|
| 単数 | `A` | 1文字の記号 |
| 複数-選択一致 | `BDE` | 記号を連結（順不同）。Bot側で昇順ソートしてAPIへ送信 |
| 複数-順番一致（重複なし） | `BDE` | 記号を順番通りに連結。入力順が当選条件。重複文字はエラー |
| 複数-順番一致（重複あり） | `BBD` | 記号を順番通りに連結。同一記号の重複入力を許可 |

オートコンプリート: ゲームを選択した後、賭け方式と選択数をヒント表示する（例: `3文字を順番通りに入力してください（例: BDE）`）。

### 処理フロー

1. ゲームIDが存在するか確認
2. ゲームが公開中かつ受付中（締め切り前）か確認
3. `option` の文字数がゲームの `requiredSelections` と一致するか確認（`single` なら1文字）
4. `option` の各文字がゲームの賭け項目に存在するか確認
5. `single` / `複数-選択一致` / `複数-順番一致（重複なし）` の場合、重複文字がないか確認
6. `複数-選択一致` の場合、`option` を昇順ソートして正規化
7. `borrow = false` の場合のみ、ユーザーの所持ポイントが `amount` 以上か確認
8. 既存の賭けがある場合は上書き（旧賭けの種別に応じてポイントまたは借金を返却）
9. `PUT /api/games/:gameId/bets` を `allowDebt` フラグ付きで呼び出して登録（upsert）

### 応答メッセージ（Ephemeral）

賭けた項目の表示形式は賭け方式に応じて変わる。

| 方式 | 表示形式例 |
|------|-----------|
| 単数 | `A: チームA` |
| 複数-選択一致 | `B: チームB、D: チームD、E: チームE（選択一致）` |
| 複数-順番一致（重複なし） | `B: チームB → D: チームD → E: チームE（順番一致・重複なし）` |
| 複数-順番一致（重複あり） | `B: チームB → B: チームB → D: チームD（順番一致・重複あり）` |

**成功時（新規・通常）**
```
✅ 賭けを受け付けました！
ゲーム: 第1試合
賭けた項目: B: チームB → D: チームD → E: チームE（順番一致・重複なし）
賭けたポイント: 50pt
現在の所持ポイント: 9,950pt
```

**成功時（新規・借金）**
```
✅ 賭けを受け付けました！（借金）
ゲーム: 第1試合
賭けた項目: B: チームB → D: チームD → E: チームE（順番一致・重複なし）
賭けたポイント: 50pt
この賭けの借金額: 50pt
現在の所持ポイント: 10,000pt（変動なし）
```

**成功時（上書き・通常）**
```
🔄 賭けを変更しました！
ゲーム: 第1試合
賭けた項目: B: チームB → D: チームD → E: チームE（BED から変更）
賭けたポイント: 100pt（50pt から変更）
現在の所持ポイント: 9,900pt
```

**成功時（上書き・借金）**
```
🔄 賭けを変更しました！（借金）
ゲーム: 第1試合
賭けた項目: B: チームB → D: チームD → E: チームE（BED から変更）
賭けたポイント: 100pt（50pt から変更）
この賭けの借金額: 100pt
現在の所持ポイント: 10,000pt（変動なし）
```

**エラー時（例）**
```
❌ 賭けに失敗しました。
理由: 締め切り時刻を過ぎています。
```

### エラー一覧

| 条件 | メッセージ |
|------|-----------|
| ゲームが存在しない | 指定されたゲームが見つかりません。 |
| ゲームが非公開または締め切り済 | このゲームは現在受け付けていません。 |
| option の文字数が不正 | X文字で入力してください。（このゲームの選択数: X） |
| option に存在しない記号が含まれる | 指定した記号が見つかりません。（無効: X） |
| option に重複文字がある（重複不可方式） | 同じ記号を複数回指定できません。 |
| ポイント不足（`borrow=false` のとき） | 所持ポイントが足りません。（所持: XXpt） |
| amount が 0 以下 | 賭けるポイントは1以上を指定してください。 |

## `/mybet` - 自分の賭け状況を確認する

### 説明
指定したゲームにおける自分の賭け状況を表示する。

### オプション

| オプション名 | 型 | 必須 | 説明 |
|------------|-----|------|------|
| `game` | Integer | ○ | ゲームID |

### 処理フロー

1. ゲームIDが存在するか確認
2. `GET /api/users/discord/:discordId` でユーザー内部 ID を取得（未登録の場合はエラー）
3. `GET /api/games/:gameId/bets` から賭け一覧を取得し、`userId` が一致するレコードを抽出

### 応答メッセージ（Ephemeral）

**賭けあり・結果未確定時（複数方式の例）**
```
📋 ゲーム: 第1試合  [複数-順番一致（重複なし） / 3択]
締め切り: 2024-01-01 12:00

あなたの賭け:
  賭けた項目: B: チームB → D: チームD → E: チームE
  賭けたポイント: 50pt
  現在の倍率: ×2.4倍（当選時: +120pt 予定）
```

**賭けあり・結果確定済み時（複数方式の例）**
```
📋 ゲーム: 第1試合（結果確定）  [複数-順番一致（重複なし） / 3択]

あなたの賭け:
  賭けた項目: B: チームB → D: チームD → E: チームE ✅ 当選
  賭けたポイント: 50pt
  獲得ポイント: +120pt
```

**賭けなし時**
```
このゲームにはまだ賭けていません。
```

**エラー時**
```
❌ 指定されたゲームが見つかりません。
```

## `/mybets` - 開催中イベントの賭け一覧を確認する

### 説明
現在開催中のイベントで自分が賭けたすべてのゲームの状況をまとめて表示する。

### オプション
なし

### 処理フロー

1. `GET /api/events` でイベント一覧を取得し、`isActive: true` のイベントを特定
2. 開催中イベントが存在しない場合はエラーを返す
3. `GET /api/users/discord/:discordId` でユーザー内部 ID を取得（未登録の場合はエラー）
4. `GET /api/users/:id/event-bets/:eventId` でイベント内の全賭け情報を取得

### 応答メッセージ（Ephemeral）

**賭けあり（混在例: 複数方式を含む）**
```
📋 **〇〇大会** - あなたの賭け一覧

第1試合 ✅ 当選  [単数]
  A: チームA に 50pt | 獲得: +70pt

第2試合 ❌ 落選  [複数-選択一致 / 3択]
  B: チームB、D: チームD、E: チームE に 100pt（借金） | 獲得: +0pt

第3試合 ⏰ 受付中  [複数-順番一致（重複なし） / 3択]
  B: チームB → D: チームD → E: チームE に 200pt | 現在倍率: ×1.8倍（当選時: +360pt 予定）

第4試合 🔒 締切済  [複数-順番一致（重複あり） / 3択]
  B: チームB → B: チームB → D: チームD に 150pt（借金） | 現在倍率: ×3.2倍（当選時: +480pt 予定）

合計 4ゲーム | 1勝 1敗 | 現在の所持ポイント: 9,720pt
```

**賭けなし時**
```
📋 **〇〇大会**
このイベントにはまだ賭けていません。
```

**エラー時**
```
❌ 現在開催中のイベントがありません。
```
```
❌ ユーザー情報が見つかりません。先に /bet を実行してください。
```

### ゲーム行のフォーマット仕様

| ゲーム状態 | アイコン | 追加情報 |
|-----------|---------|---------|
| `open`（受付中） | ⏰ | 現在倍率・当選時獲得予定ポイント |
| `closed`（締切済） | 🔒 | 現在倍率・当選時獲得予定ポイント |
| `finished` 当選 | ✅ | 獲得ポイント |
| `finished` 落選 | ❌ | 獲得: +0pt |

借金賭けの場合は項目名の後に `（借金）` を付記する。

## `/admin-link` - 管理者用 WebアプリのURLを表示する

### 説明
管理者が Web アプリにアクセスするための URL（トークン付き）を表示する。

### オプション
なし

### 処理フロー

1. 管理者ロールを確認（なければ拒否）
2. 環境変数からトークンと WebアプリのベースURLを取得
3. ハッシュフラグメント内にトークンを埋め込んだホーム画面の URL を生成して返信

### 応答メッセージ（Ephemeral）

```
🔑 管理者用URLです。他の人に共有しないでください。

https://<github-pages-url>/#/?token=xxxxxxxxxxxxxxxx
```

トークンはハッシュ内クエリパラメータとして付与するため、サーバーのアクセスログには残らない。

## `/post-game` - ゲーム情報をチャンネルに投稿する

### 説明
指定したゲームの情報（タイトル・賭け項目・締め切り）を現在のチャンネルへ全員向けに投稿する。

### オプション

| オプション名 | 型 | 必須 | 説明 |
|------------|-----|------|------|
| `game` | Integer | ○ | ゲームID |

### 処理フロー

1. 管理者ロールを確認（なければ拒否）
2. `GET /api/games/:id` でゲーム情報を取得
3. ゲームが公開中か確認
4. 整形してチャンネルへ送信（全員表示）

### 投稿メッセージ（全員表示）

**単数方式の場合**
```
🎮 **ゲーム情報** - 第1試合

説明: チームAとチームBの対戦です。

賭け項目:
  A: チームA
  B: チームB

締め切り: 2024-01-01 12:00

賭けるには `/bet game:1 option:<記号> amount:<ポイント>` を使ってください。
例: `/bet game:1 option:A amount:100`
```

**複数方式の場合（例: 複数-順番一致 / 3択）**
```
🎮 **ゲーム情報** - 第1試合  [複数-順番一致（重複なし） / 3択]

説明: 1位・2位・3位を当ててください。

賭け項目:
  A: チームA
  B: チームB
  C: チームC
  D: チームD
  E: チームE

締め切り: 2024-01-01 12:00

賭けるには `/bet game:1 option:<記号を3文字> amount:<ポイント>` を使ってください。
例: `/bet game:1 option:BDE amount:100`（B→D→E の順で予想）
```

### エラー時（Ephemeral）

| 条件 | メッセージ |
|------|-----------|
| ゲームが存在しない | 指定されたゲームが見つかりません。 |
| ゲームが非公開 | 非公開のゲームは投稿できません。 |
| 管理者権限なし | このコマンドは管理者のみ使用できます。 |

## 環境変数

| 変数名 | 説明 |
|--------|------|
| `DISCORD_BOT_TOKEN` | Discord Bot のトークン |
| `DISCORD_CLIENT_ID` | Discord アプリケーション ID |
| `DISCORD_GUILD_ID` | 対象の Discord サーバー ID |
| `DISCORD_ADMIN_ROLE_ID` | 管理者ロールの ID（複数があり得る） |
| `WEB_API_BASE_URL` | Webサーバーのベース URL |
| `WEB_APP_BASE_URL` | Webアプリのベース URL |
| `ADMIN_TOKEN` | 管理者用トークン（WebアプリのURL付与用） |
