■導線・画面名について
パンくずリストの、ホーム=イベント一覧, [イベント名]=ゲーム一覧, [ゲーム名]=ゲーム状況 にする

合わせて、画面名の変更
    イベント一覧 => ホーム画面
    ゲーム一覧 => イベント画面
    ゲーム状況 => ゲーム画面
以降、変更後の画面名で表記する

導線の変更
    ホーム画面のイベント一覧内の「編集」は、イベント画面のタイトルの横に移動する
    ホーム画面のイベント一覧内の「削除」は、上記「編集」の横に移動する
    イベント画面のゲーム一覧内の「編集」は、ゲーム画面のタイトルの横に移動する
    イベント画面のゲーム一覧内の「削除」は、上記「編集」の横に移動する

ホーム画面内のイベント一覧の「ゲーム一覧」は、「詳細」にする
イベント画面内、ゲーム一覧の「状況」は、「詳細」にする

■イベントの開催や公開非公開について
    ホーム画面のイベント一覧の開催状態は開催中・ ー の2種類の状態に変更

    イベントにも公開/非公開の概念を追加する。
    　開催された場合は、公開状況は強制的に公開に変更する。開催されている場合は、非公開設定不可。

    ホーム画面のイベント一覧の「開催切り替え」は、開催中に押下できなくなっているが、押下できてOK。つまり開催０件はあり得る。

■追加・編集画面について
    イベント編集画面が左に寄っているのが気になる。他と合わせて欲しい

    ゲーム編集画面が左に寄っているのが気になる。他と合わせて欲しい

    ゲーム編集画面にてエラー内容が上に表示されてもスクロールしないので分からない
        エラー内容は、入力項目の下に表示するようにするか、
        あるいは、エラー内容が表示されたときにスクロールして見えるようにする

    ゲーム編集画面のゲーム追加時の締め切り日時の初期値は現在時刻にする

    ラベルの修正は許可していたが、それもやはり禁止する。
    修正する場合は、非公開に戻させる。

    また、非公開への変更は、賭けが１件でもる場合は不可にする。

■イベント画面について
イベント画面内、ゲーム一覧の「公開」列は一般ユーザーには見えないようにする

■動作について
ページ遷移時にモダンなアニメーションが入るようにCSSを設定してください。

■（大きな変更）ギルドIDの追加が必要
    ギルド別にイベントやゲームを管理する必要があるため、イベントとゲームのデータ構造にギルドIDを追加する必要がある。
    これに伴い、APIのリクエストパラメータやレスポンスのデータ構造も変更する必要がある。
    さらに、Webアプリのルーティングや画面表示も、ギルドIDに基づいて適切にイベントやゲームをフィルタリングして表示するように変更する必要がある。
    ホーム画面のURLは以下のようになる
    http://127.0.0.1/#/events/
    ↓
    http://127.0.0.1/#/events/{guild_id}
    合わせて、ヘッダのタイルは「{サーバー名} ギャンブルBOT管理アプリ」とする
    これにより、DiscrodのURL表示コマンドもサーバーIDを付与したものにする必要がある
    また、一般ユーザー向けのURL表示コマンドを開放する必要がある。一般ユーザーの場合はtokenなしのURLとなる。
    

■ゲームの削除についての処理について検討が必要
- 公開したらゲーム締め切りまで削除できないようにする。
　= ユーザーのポイント不整合を防ぐため。
- ゲーム締め切り後は削除できるようにする。
　= 管理者の誤操作や、過去イベントの整理のため。
- ゲーム削除時はユーザーのポイント履歴へ一括削除を行う。
 => ポイント履歴の削除が良いのか、差額の登録が良いのかは要検討。
