# 概要設計

## 目的
Discord Bot の形態でユーザーらが所持ポイントを元に賭け(ゲーム)を行えるインターフェースと、ゲームの状況やゲーム自体を管理するWebアプリを構築することを目的とする。

## 主な内容
1. **構成**: 
- Dicord Botサーバー: ユーザーがポイントを賭けるためのインターフェースを提供します。
- DBサーバー: ユーザーのポイントやゲームの状況を保存し情報を提供します。
- Webサーバー: ゲームの管理やユーザーの賭け状況を表示するためのWebアプリやAPIを提供します。

2. **データフロー**: 
Webアプリ（ブラウザ）
    Webアプリの取得                => Webサーバー
    ゲーム作成・編集・削除          => Webサーバー
    ゲーム状況の表示                => Webサーバー
Discord クライアント
    ユーザーのポイント賭け          => Dicord Botサーバー
    ユーザーの掛けの表示            => Dicord Botサーバー
Dicord Botサーバー
    ユーザーのゲーム情報の作成・編集 => Webサーバー
    ユーザーのゲーム情報の取得      => Webサーバー
Webサーバー
    ゲームの作成・編集・削除        => DBサーバー
    ゲーム状況の取得                => DBサーバー
    ユーザーの賭けの作成・編集・削除 => DBサーバー
    ユーザーの賭けの取得            => DBサーバー

3. **主要な機能**: 
- Webアプリ: 
    - ホーム画面および基本構成: 
        - ヘッダー: ロゴ、権限
        - 左のサイドバー: イベント一覧、ユーザー管理、ポイント管理などのメニュー
        - 右のエリア: ゲーム一覧や編集画面などの詳細な内容を表示、パンくずリストも表示
        - 権限について: URLパラメータに付与されたトークンが正しければ管理者権限を付与、そうでなければ一般ユーザー権限
        - モバイルでも表示可能なレスポンシブデザイン
    - ゲームを取りまとめた「イベント」一覧表示する、イベント一覧: 
        イベントの編集への遷移（作成も兼務）（管理者）
        イベントの削除（管理者のみ表示）
        現在の開催中のイベントを切り替え（管理者のみ表示）
        イベントからゲーム一覧表示への遷移
    - イベント内の賭け「ゲーム」一覧表示する、ゲーム一覧: 
        - ゲームの編集への遷移（作成も兼務）（管理者）
        - ゲームの削除（管理者のみ表示）
        - ゲームの公開・非公開の切り替え（管理者のみ表示）
        - ゲームの賭け状況の表示する、ゲーム状況: 
            - ユーザーごとの賭けの状況
            - 賭けの締め切り時間
            - 賭ける各項目の現在の当選時の倍率表示
            - ゲームの結果の表示
        - ユーザーのゲーム推移とポイントの表示する、ユーザー結果一覧: 
            - ユーザーのポイントや借金の推移
            - ユーザーのポイントの現時点の最終ポイントと借金残高
            - ユーザーのゲームの勝敗の推移
- Discord Bot: 
    - ユーザーのポイント賭け(ゲーム番号, 賭ける項目, 賭けるポイント数、借金するか)
    - ユーザーの掛けの表示(ゲーム番号)
    - （管理者のみ）権限で入れるWebアプリへリンクの表示(トークンをURLパラメータに付与)
    - （管理者のみ）ゲーム内容をDiscordのチャットに書き込む(ゲームタイトル、賭け項目、締め切り)

4. **技術スタック**:
- プログラミング言語: TypeScript
- フレームワーク: 
    - Webサーバー: Express.js
    - Webアプリ: React.js
    - Discord Botサーバー: discord.js
- データベース: PostgreSQLを使用予定
- インフラ: 
    - Webサーバー: GitHub Pagesを使用予定
    - DBサーバー,
      Webサーバー,
      Discord Botサーバー:
      個人用デスクトップPCでの運用を想定しているため、ローカル環境でDockerを使用して各サーバーをコンテナ化し、Docker Composeで管理予定

5. **非機能要件**: 
- セキュリティ: ユーザーデータやゲームデータの保護、管理者権限の適切な管理
- パフォーマンス: ユーザーの賭けやゲームの進行がスムーズに行えるようなレスポンスの確保
- スケーラビリティ: 将来的なユーザー数の増加や機能追加に対応できる設計
- メンテナンス性: コードの可読性やモジュール化を考慮した設計
- ロギングとモニタリング: エラーや重要なイベントのログを適切に記録し、システムの状態を監視する仕組みの導入
- 運用想定：個人用デスクトップPCでの運用を想定。リソースの制約がある点について考慮が必要。

6. **開発プロセス**:
- バージョン管理: Gitを使用し、GitHubなどのリモートリポジトリでコードを管理
- ブランチ戦略: フィーチャーブランチを使用し、コードレビューを行う
- コミット規約: 意味のあるコミットメッセージを使用し、変更内容を明確にする
- テスト: ユニットテストや統合テストを実装し、コードの品質を確保する
- ドキュメント: コードのドキュメントやAPIドキュメントを整備し、開発者が理解しやすいようにする
- デプロイ: Docker Composeを使用して、ローカル環境でのデプロイを簡単に行えるようにする
